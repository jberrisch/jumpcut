Jumpcut 0.81 corrects a number of regressions introduced during the 0.80 rewrite:

  • We restore the Jumpcut bezel's ability to appear in any space when using multiple desktops through Mission Control.
  • Jumpcut no longer crashes when "Clear All" is selected but there are no clippings on the stack.
  • Per multiple users' requests, Jumpcut can again be with in-memory clippings only, with no save file committed to disk. (This feature must be turned on using Apple's defaults command-line application.)
  • An option is now exposed allowing a choice between the bezel following the last-used clipping as new clippings are added or moving to the top. This fixes a bug where the bezel neither moved to the top nor followed the last used clip but instead stayed at the same position throughout as clippings changed.
  • After user feedback, we restore jumping to clippings in the bezel using the number keys. Additionally, "Home", "End", "Page Up", and "Page Down" have been restored to the Jumpcut 0.75 behavior.
  • The "remembering" stack size preference is now honored (although only checked at startup).
  • We've made additional tweaks to bezel keydown handling, which was being applied inconsistently for non-QWERTY keyboards. (Thanks to GitHub user Vadorequest!)
  • Finally, per a user request, the "white scissors" emoji is once again an optional status bar icon.

As always, please report any bugs or feature requests on GitHub. Thanks for using Jumpcut!
